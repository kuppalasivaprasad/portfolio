04 · Projects

Case studies from the field.

Azure · CloudOps

Enterprise Azure Platform & AKS Lifecycle Management

Overview

Ongoing management of AT&T's Azure footprint at Tech Mahindra — infrastructure, security posture, and Kubernetes lifecycle.

Problem

Enterprise Azure environments need strict access control, network segmentation, and AKS upgrades that don't disrupt live workloads.

Responsibilities

Managed VMs, VNets, and storage; enforced RBAC/PIM policies; configured Azure Firewall segmentation; built golden images with Packer; planned and validated AKS upgrades.

Architecture

Azure Firewall governs traffic between segmented VNets, Azure Monitor Agent feeds observability, and Packer-built golden images standardize compute across environments feeding into AKS.

Business Impact

Strengthened security and compliance posture while keeping AKS upgrades non-disruptive to production traffic.

AzureAKSRBAC/PIMPacker
AWS · Kubernetes

AWS Cloud-Native Platform & Pipeline Automation

Overview

Built and operated AWS infrastructure for Dish Network (Sling) using Terraform, EKS Fargate, and on-prem Rancher clusters.

Problem

The team needed repeatable infrastructure-as-code, containerized workloads across hybrid environments, and safer release pipelines.

Responsibilities

Provisioned EC2, ALB, Lambda, Step Functions, VPC, Route 53, IAM, S3, and EKS Fargate via Terraform; managed GitLab CI runners; wrote Helm charts; led EKS upgrade planning.

Architecture

Terraform-provisioned VPC and EKS Fargate profiles run containerized services alongside on-prem Rancher clusters, fronted by ALB and orchestrated through gated/one-click GitLab CI pipelines.

Business Impact

Gated and one-click pipelines streamlined releases; phased EKS upgrade strategy kept downtime to a minimum during cluster upgrades.

TerraformEKS FargateHelmGitLab CI
Release Engineering · Compliance

Release Automation for a Regulated Document Platform

Overview

DevOps and change coordination for Bank of America's Compliance Matter Document Management platform at Infosys.

Problem

A regulated financial platform needed auditable, automated releases and proven disaster-recovery readiness.

Responsibilities

Built Jenkins pipelines; managed Bitbucket branching and PR review; configured XLR for release automation; integrated Ansible with Jenkins; scheduled Autosys batch jobs; monitored with Dynatrace and Splunk.

Architecture

XLR orchestrates release workflows triggered from Jenkins pipelines, with Ansible handling automated deployment steps and Autosys managing scheduled batch/purge jobs downstream.

Business Impact

Ansible-Jenkins integration reduced deployment times; ARC fail-over/fall-back exercises validated resiliency for a compliance-critical system.
